# 完整分析流程（full-analysis）

端到端工作流：调研 → 分析 → 生成PDF报告。

## 触发条件

用户说"高考志愿"/"报志愿"/"选大学"/"选专业"等关键词时触发。

## 流程

### 1. 检查已有数据

```
检查工作目录是否有 student.json
├─ 有 → 询问"发现已有考生档案(姓名XX)，是否复用？"
│   ├─ 复用 → 跳到Step 3
│   └─ 重新调研 → 继续Step 2
└─ 无 → 继续Step 2
```

### 2. 考生调研（survey.py）

选择调研模式：
- **模式A：对话问答** — AI逐个提问30+问题，用户在对话中回答
- **模式B：网页表单** — 生成HTML表单，用户在浏览器填写后提交

#### 30+问题6个维度

| 维度 | 题数 | 核心 |
|---|---|---|
| 基本信息 | 6 | 姓名/性别/省份/分数/位次/选科 |
| 学科能力 | 6 | 各科分数 |
| 性格职业 | 6 | MBTI简化版4题 + 霍兰德2题 |
| 家庭环境 | 5 | 经济/学历/地域偏好/学费预算 |
| 就业意向 | 5 | 考研/考公/企业类型/行业/排除项 |
| 健康限制 | 4 | 色觉/视力/身高/其他 |

输出：`student.json`

### 3. 加载数据

```
检查是否有 data_{省份}_{年份}.json
├─ 有且当年 → 直接使用
├─ 有但往年 → 提示数据较旧，询问是否继续
└─ 无 → P0使用内置数据包 / P1+调用scraper.py爬取
```

同时加载政策模板：`references/policy-templates/{省份}.md`

### 4. 运行分析引擎（analyzer.py）

6步分析框架（详见 `references/methodology.md`）：

```
Step 1: 学科优势分析 → 识别王牌/软肋
Step 2: 方向筛选 → 推荐6-8个方向
Step 3: 院校匹配 → 冲/稳/保三档
Step 4: 就业评估 → 附就业数据（来自major-database.md）
Step 5: 96志愿排序 → 冲10/稳56/保30
Step 6: 风险评估 → 滑档/退档防范
```

输出：`analysis.json`

### 5. 生成报告

选择输出格式：
- PDF（默认）：调用 `scripts/pdf_generator.py`
- Excel：调用 `scripts/excel_generator.py`（P2）
- 网页：调用 `scripts/web_generator.py`（P2）

PDF生成流程：
```
analysis.json → render_html() → temp_report.html → Playwright → output.pdf
```

报告交付到桌面。

### 6. 报告内容结构

```
封面（考生信息+学科成绩）
一、评估范围与筛选标准
二、学科优势分析
三、推荐方向（6-8个，含匹配度）
四、各方向详情（院校+就业+课程+门槛）
五、冲档备选
六、96志愿清单（冲/稳/保）
七、避免滑档退档指南
```

## 快速建议模式（quick-advice）

只需3个信息即可快速出建议：
1. 分数+位次
2. 选科组合
3. 省份

跳过深度调研，直接给方向建议+院校清单。适合时间紧迫的场景。

## 注意事项

- 所有数据标注来源URL+采集日期
- 色觉确认是填报前最紧急的事
- 96个志愿全部勾选"专业服从调剂"
- 保底位次必须 > 考生位次 × 1.5
